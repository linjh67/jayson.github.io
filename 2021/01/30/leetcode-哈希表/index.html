哈希表

基于hash函数H，根据此函数和查找关键字key，直接确定查找值所在的位置

通过hash把key转换成一个整型数字，然后将该数字对数组长度取余，取余结果为数组下标，将key对应的value存储在该下标数组空间中

牺牲了空间换取时间。

用途

判断一个元素是否出现在集合里

哈希冲突

拉链法

在存储数据后面加一个指针，指向后面冲突的数据

线性探测法

利用哈希表空余的位置来解决碰撞，碰撞的时候根据不同的strategy向左或向右寻找可用的地址。

常见的hash结构

数组

set

std::set

底层实现：红黑树，有序，数值不可以重复，不能改变数值，查询效率O(logn),增删效率O(logn)

std::multiset

底层实现：红黑树，有序，数值可以重复，不能改变数值，查询效率O(logn),增删效率O(logn)

std::unordered_set

底层实现：哈希表，无序，数值不可以重复，不能改变数值，查询效率O(1),增删效率O(1)

拥有快速的查找和删除，添加的优点，没有红黑数自动排序的功能。

题目描述

编写一个算法来判断一个数 n 是不是快乐数。

「快乐数」定义为：对于一个正整数，每一次将该数替换为它每个位置上的数字的平方和，然后重复这个过程直到这个数变为 1，也可能是 无限循环 但始终变不到 1。如果 可以变为 1，那么这个数就是快乐数。

如果 n 是快乐数就返回 True ；不是，则返回 False 。

解题思路

排序+ 双指针法

此题的关键在于去重，所以先将其排序简化操作，然后先确定a，再确定b，c，使得a+b+c=0；

  1. a的确定

a=nums[i],

假如a>0，则结束

假如前面出现过一次相同的a，即nums[i]=nums[i-1]，不予操作，达到去重效果。

  1. b,c的确定

b,c分别是$left,right$，根据$sum=nums[i]+nums[left]+nums[right]$的值确定right，left的移动

去重的话则是假如$sum=0$的时候，$nums[right]=nums[right-1]$的话就将$right$左移，$left$同理。

哈希表法

这题不是很适合用哈希表法,但也是先确定a。

然后b，c根据哈希表来做，即找哈希表里是否有$-(nums[i]+nums[j])$,没有的话插入$nums[j]$
